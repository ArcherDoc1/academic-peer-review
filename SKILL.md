# 学术论文同行评审

## 目标

以审稿人视角检查论文的科学性、透明度、可重复性、论证边界与报告质量。将每项批评绑定到稿件中的具体证据，并给出作者可以执行的修改建议。

## 工作流程

1. 完整读取可用稿件、补充材料和用户提供的期刊要求。保留页码、行号、章节、表格和图号等定位信息。
2. 判断稿件类型：
   - 原创研究、实验研究、观察性研究、建模或方法研究：读取 `references/original-research.md`。
   - 叙述性综述、系统综述、范围综述或证据综述：读取 `references/review-article.md`。
   - 含荟萃分析、元回归或定量证据综合：读取综述检查表，并额外执行其中的统计分析检查。
   - 类型不明确或混合时，说明判断依据，并组合相关检查项。
3. 建立证据表。对每个潜在问题记录：位置、稿件原文或准确转述、问题、影响、建议。缺少全文或附件打不开时，明确评审边界，不猜测缺失内容。
4. 先判断问题是否成立，再分级：
   - `关键问题`：影响研究有效性、可重复性、主要结论或科研诚信，通常需要实质修改。
   - `一般问题`：影响清晰度、完整性或报告规范，但不直接推翻主要结论。
   - `建议`：可提升可读性、检索性或表达质量的非必要优化。
5. 按 `references/review-report-template.md` 组织审稿报告。除非用户指定其他格式，先列关键问题，再列一般问题；不要按阅读顺序堆砌零散意见。
6. 完成质量闸门后再输出。

## 证据与推理规则

- 只评价实际可见的稿件内容。不要臆造数据、页码、行号、引用、作者意图或未提供的期刊政策。
- 区分三类表述：稿件明确报告的事实、从稿件可合理推出的判断、需要作者澄清的未知信息。
- 每条关键问题尽量包含“位置 → 观察 → 为什么重要 → 如何修改”。无法精确定位时至少给出章节和相关主题。
- 检查文字、表格、图、摘要和结论之间的一致性；不要把“未在正文发现”写成“作者没有做过”。
- 不以个人偏好替代科学或报告规范。要求新增实验、分析或引用时，说明其必要性和对结论的影响。
- 不要求作者引用审稿人本人或无关工作。需要外部核验时优先使用原始论文、官方指南或期刊政策，并把外部事实与稿件内证据分开。
- 对伦理、数据完整性、图像异常、抄袭或其他科研诚信风险使用审慎措辞，陈述可观察信号并建议编辑部核查，不在证据不足时直接定性。
- 保持专业、具体、尊重作者。批评研究和报告，不评价作者个人。

## 评审深度

- 用户只要求快速意见时，输出简要总结、3 至 5 条最重要问题和总体判断。
- 用户要求完整审稿时，覆盖相应检查表的全部栏目，并报告“符合”“部分符合”“不符合”或“无法判断”。
- 用户要求逐段、逐页或逐行评审时，增加定位和局部修改建议，但仍在开头汇总最重要的科学问题。
- 用户未提供目标期刊或评分尺度时，不虚构期刊标准；采用通用学术质量标准，并说明适配期刊范围尚未评估。
- 只有用户要求决定或提供了期刊选项时，才给出“接收/小修/大修/拒稿”等推荐。说明推荐取决于当前稿件与已知标准。

## 输出要求

- 使用用户的语言；用户要求英文审稿时使用自然、克制的学术英语。
- 先给一段准确的研究概述，证明已理解论文，再提出意见。
- 关键问题使用编号；每条只处理一个主要问题。一般问题保持简短并定位。
- 对无法判断的项目说明还需要哪些材料或信息。
- 如用户要求作者回复清单，将每条意见改写成可逐项回应的问句或行动项。
- 如用户要求评分，先复述评分维度和量表，再逐项给出分数及一句证据依据。

## 完成前质量闸门

确认以下事项：

- 已正确识别论文类型并加载对应参考文件。
- 已检查摘要、正文、图表、结论和参考文献之间的重要不一致。
- 每条关键问题都有稿件依据、影响说明和可执行建议。
- 已区分稿件缺陷、信息缺失和审稿人无法访问的材料。
- 未捏造引用、数据、统计结果、伦理违规或期刊要求。
- 总体建议与前述问题严重程度一致。
