# 审稿报告模板

根据用户要求删减栏目。不要输出空标题；无法判断时说明缺少的材料。

## 稿件信息与评审范围

- 识别的稿件类型：
- 可用材料：
- 未提供或无法访问的材料：
- 采用的期刊或评价标准：

## 一、研究概述

用一段话准确说明研究问题、设计或综述类型、数据或证据来源、主要结果和作者结论。不要照抄摘要，也不要在此处提前堆砌批评。

## 二、总体评价

概括论文的主要贡献、最重要优点和决定当前评价的核心不足。控制在一至三段。

## 三、关键问题

每条使用以下结构：

### 1. 简短问题标题

- **位置：** 页码/行号/章节/表格/图号。
- **观察：** 稿件实际报告、遗漏或不一致之处。
- **影响：** 该问题为什么会影响有效性、可重复性、解释或主要结论。
- **建议：** 作者需要补充、重做、澄清、降级结论或讨论的具体行动。

## 四、一般问题

使用编号短项。包含必要定位和直接修改建议；一个条目只讨论一个问题。

## 五、分章节检查（完整评审时使用）

按对应检查表逐项报告：标题、摘要、引言、方法、结果、讨论、参考文献和总体事项；综述论文改用相关性、历史与前瞻、证据覆盖、方法、统计、综合与报告质量。

建议采用：`符合 / 部分符合 / 不符合 / 无法判断 — 证据与说明`。

## 六、伦理、完整性与专业复核

- 伦理审批与知情同意：
- 数据、图像或报告完整性信号：
- 利益冲突与透明度：
- 是否建议统计学、方法学或其他专业审查及原因：

## 七、给编辑的建议（仅在用户要求时）

- 推荐：接收 / 小修 / 大修 / 拒稿 / 无法判断。
- 核心依据：用两至四句连接推荐与前述问题严重程度。
- 保密事项：只放不适合发送给作者、且有明确依据的编辑部核查建议。

## 八、作者回复清单（用户要求时）

把关键问题和一般问题转换成可逐项回应的行动项，例如：“请在方法部分补充……并说明……”。
