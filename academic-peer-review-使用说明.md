# academic-peer-review 使用说明

适用对象：需要对原创研究论文、综述论文或含荟萃分析的稿件进行同行评审的作者、审稿人、编辑与研究团队。

## 1. 这个 Skill 能做什么

`academic-peer-review` 会先识别稿件类型，再使用相应检查表开展评审：

- 原创研究：重点检查摘要、引言、方法、统计、结果、讨论、参考文献、伦理与可重复性；
- 综述论文：重点检查综述必要性、证据覆盖、检索与筛选透明度、偏倚风险、结果综合、报告规范与语言质量；
- 含荟萃分析的综述：除综述检查外，额外检查效应量、模型选择、异质性、敏感性分析和结果解释。

输出不只是“好或不好”，而是把意见组织为“位置—观察—影响—建议”，并按关键问题、一般问题和优化建议分级。

## 2. 开始前准备

优先提供以下材料，评审会更准确：

- 论文全文，包含图表和参考文献；
- 补充材料、研究方案、预注册信息或审稿表（如有）；
- 稿件类型：原创研究、叙述性综述、系统综述、范围综述或荟萃分析；
- 目标期刊、期刊审稿标准或评分量表（如有）；
- 希望的输出语言、评审深度和用途，例如投稿前自查、正式审稿或作者回复。

没有提供期刊要求时，Skill 会采用通用学术质量标准，并明确哪些事项无法判断。

## 3. 最快使用方式

将稿件上传到当前会话后，直接输入：

```text
使用 $academic-peer-review 对这篇论文进行完整同行评审。请先识别稿件类型，再按关键问题、一般问题和分章节检查输出；每条关键问题都请注明位置、影响和可执行的修改建议。
```

如果只想先找最重要的问题：

```text
使用 $academic-peer-review 快速评审这篇稿件。只输出研究概述、最重要的 5 个问题、每个问题的修改建议，以及总体判断。
```

## 4. 常用指令模板

### 原创研究论文

```text
使用 $academic-peer-review 评审这篇原创研究论文。重点检查研究设计、样本量与抽样、控制变量、统计方法、效应量和置信区间、图表与正文一致性、局限性，以及结论是否超出数据支持范围。
```

### 系统综述或范围综述

```text
使用 $academic-peer-review 评审这篇系统综述。重点检查研究问题与综述必要性、检索数据库和检索式、纳入排除标准、筛选流程、偏倚风险、数据提取、证据综合，以及是否符合适用的报告规范。
```

### 含荟萃分析的综述

```text
使用 $academic-peer-review 评审这篇含荟萃分析的系统综述。除常规综述评审外，请重点检查效应量、模型选择、异质性、发表偏倚、亚组与敏感性分析、置信区间和结论边界；如有必要，说明是否建议统计学专家复核。
```

### 英文期刊审稿意见

```text
Use $academic-peer-review to write a concise journal-style peer-review report in English. Separate major and minor comments, cite manuscript sections or page numbers, and give actionable recommendations without inventing data or references.
```

### 投稿前自查

```text
使用 $academic-peer-review 做投稿前自查。请按“符合、部分符合、不符合、无法判断”逐项检查，并优先列出会影响研究有效性、可重复性或主要结论的问题。
```

### 生成作者回复清单

```text
使用 $academic-peer-review 审查稿件后，把所有意见转换成作者可逐条回应的行动清单。每条都说明需要补充、澄清、修改或降级结论的具体位置。
```

## 5. 如何指定评审深度

- **快速评审**：研究概述、3 至 5 条最重要问题和总体判断，适合初筛或讨论。
- **完整审稿**：覆盖对应检查表全部栏目，适合投稿前评估或正式同行评议。
- **逐章节评审**：按摘要、引言、方法、结果、讨论等章节给出具体意见。
- **逐页或逐行评审**：适合已有页码或行号的定稿；请明确要求保留定位。
- **评分评审**：先提供量表或维度，例如创新性、方法学严谨性、清晰度和临床意义；Skill 会逐项评分并说明依据。

## 6. 默认输出结构

完整评审通常包含以下部分：

1. 稿件类型、可用材料与评审范围；
2. 研究概述；
3. 总体评价：主要贡献、优点与决定性不足；
4. 关键问题：位置、观察、影响、修改建议；
5. 一般问题：简短、可定位、可执行；
6. 分章节检查；
7. 伦理、数据完整性、利益冲突与是否需要专业复核；
8. 给编辑的建议或作者回复清单（仅在明确要求时输出）。

若未提供目标期刊或决定选项，Skill 不会擅自给出“接收/拒稿”等编辑决定。

## 7. 获得更好结果的提示

- 明确你希望评审的目标，例如“找会导致大修的问题”或“重点看统计”。
- 提供图、表和补充材料；仅有摘要时只能进行有限评审。
- 有目标期刊时，把期刊范围、审稿表和格式要求一并提供。
- 要求精确定位时，提供带页码或行号的版本，并在指令中要求“标注页码/章节/图表号”。
- 如需外部事实核验、指南符合性或文献更新，请明确授权检索；不要把外部核验与稿件内部评价混为一谈。

## 8. 使用边界

- Skill 只依据当前可访问的稿件和材料评审，不会猜测未提供的数据、结果、页码或参考文献。
- “未在文中发现”不等于“作者没有做过”；这类事项会被表述为需要澄清或补充报告。
- 发现伦理、图像、数据完整性或科研诚信风险时，Skill 只描述可观察信号并建议编辑部核查，不会在证据不足时直接定性。
- Skill 可以辅助评审，但不能替代领域专家、统计学专家、伦理委员会或期刊编辑的最终判断。

## 9. 常见问题

### 为什么没有给出接收或拒稿建议？

默认不会给出编辑决定。请在指令中明确要求，并提供期刊选项或评价标准，例如：“请在接收、小修、大修、拒稿中给出推荐，并说明依据。”

### 为什么有些项目显示“无法判断”？

通常是因为缺少全文、补充材料、方案、代码、注册信息或期刊要求。这是有意保留的不确定性，不是遗漏。

### 为什么意见太宽泛？

要求“完整审稿”“逐章节检查”或“每条关键问题必须包含位置、影响和修改建议”，并提供带页码或行号的论文版本。

### 可以只评审统计部分吗？

可以。直接说明研究类型和统计关注点，例如模型选择、缺失值、效应量、置信区间、异质性或敏感性分析。

## 10. 保密与合规

处理未公开稿件前，请确认使用环境符合期刊保密要求、机构数据政策和作者授权范围。Skill 本身不包含上传或执行代码，但所用运行时或模型服务的隐私规则仍由使用者负责确认。
